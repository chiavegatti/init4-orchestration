## What was done
Describe objectively what was implemented or generated by the AI.

Focus on facts, not opinions.

---

## Why it was done
Explain the goal or instruction that led to this implementation.

---

## Issues encountered
Describe what was wrong, incomplete or incorrect.

Examples:
- incorrect assumption
- missing constraint
- wrong interpretation of schema or API
- hallucinated behavior

---

## Resolution
Describe how the issue was corrected.

Be explicit:
- what changed
- what rule was enforced
- what the correct behavior is

---

## Lessons learned
Write **short, actionable rules**.

Good examples:
- "Never assume X without explicit schema."
- "Always validate Y against Z before implementation."
- "This pattern requires an ADR."

Bad examples:
- Vague opinions
- Complaints
- Emotional statements
